ArchLinux 扩展副屏

  为了体验鸿蒙系统上车华为平板，平时刷刷剧、看看书也是蛮不错的。最近总想着多屏幕耍电脑(之前买的DIY显示器坏了)，看着最近吃灰的平板，又是想着折腾 sunshine + moonlight (逛 github 时看到 MoonlightOH)。在 MoonlightOH 发现 Moonlight 已经适配上车，于是果断拿出平板开始折腾。

  整个流程简单来说，识别额外的显示屏幕(诱骗器或软件模拟显示器)然后主机通过 sunshine 串流发送，客户端 moonlight 接收显示。当然其功能远远不止如此，不过对于我来说，作为副屏使用恰到好处(又省下一笔显示器费用😁)。

一、环境

  • Arch Linux

  • HUAWEI MatePad Pro 13.2(2024)-PCE-W30

    • HarmonyOS 5.1.0

  • 绿联 HDMI欺显卡骗器虚拟显卡

二、安装 sunshine 与 Moonlight

这里没什么大问题，都是现成的资源：

  • sunshine
  • moonlight

直接在平板的应用商店搜索安装即可。

四、问题

  看起来很简单，但是我还是折腾了好几个小时，周五从11点搞到1点半，受困于 capture 问题，周六晚上又折腾2个小时，1点半左右才弄好。弄完心情大爽！

Q1. wayland 启动 capture 异常：Unable to initialize capture method
A1. X11 之前启动时并没有这个问题，这里的问题是权限不足。给 Sunshine 添加权限 sudo setcap cap_sys_admin+p $(readlink -f $(which sunshine)) 重启 Sunshine 即可。

Q2. Unable to initialize capture method Platform failed to initialize
A2. 添加权限后发现没有找到任何可用的显示输出(此时幸好我显卡诱骗器带EDID)，如果没有 EDID 的话，可以看下其他博客。强制KMS捕获 capture = kms

Q3. Error: Failed to create client: Daemon not running(Sunshine 的后台守护进程没起来)
A3. systemctl enable avahi-daemon

Q4. 防火墙问题

五、结语

  目前体验 moonlight 客户端使用 1080P-60HZ/120HZ 没什么问题，不过当开启2K以上高码率时，发现存在画面撕裂的现象，使用 1080P-60HZ 延迟超低，仅2-3ms。(完了，生活作息乱了😭)
