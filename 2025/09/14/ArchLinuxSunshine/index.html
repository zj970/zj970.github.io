ArchLinux 扩展副屏

  为了体验鸿蒙系统上车华为平板，平时刷刷剧、看看书也是蛮不错的。最近总想着多屏幕耍电脑(之前买的DIY显示器坏了)，看着最近吃灰的平板，又是想着折腾 sunshine + moonlight (逛 github 时看到 MoonlightOH)。在 MoonlightOH 发现 Moonlight 已经适配上车，于是果断拿出平板开始折腾。

  整个流程简单来说，识别额外的显示屏幕(诱骗器或软件模拟显示器)然后主机通过 sunshine 串流发送，客户端 moonlight 接收显示。当然其功能远远不止如此，不过对于我来说，作为副屏使用恰到好处(又省下一笔显示器费用😁)。

二、安装 sunshine 与 Moonlight

这里没什么大问题，都是现成的资源：

  • sunshine
yay -S sunshine
  • moonlight

直接在平板的应用商店搜索安装即可。

三、配置

  由于我之前折腾过显卡直通(想玩LOL下棋来着)所以就买了显卡诱骗器，这玩意不贵也挺好用的。这次它就减少了我折腾的很大时间，最起码 EDID 不用伪装了。不过一些便宜的是不带 EDID 信息，第一个买的 8 元的就是不带 EDID 的，显卡直通直接失败(没退货血亏)。

  回归主题，当安装完 sunshine 后，启动服务并运行程序，即可在 sunshine web ui 界面进行初步配置：https://localhost:47990/， 首次需要配置注册账号密码。配置这里建议大家多读官方文档说明。点击跳转

# 打开开机自启动 sunshine
systemctl --user enable sunshine
# 关闭开机自启动 sunshine
systemctl --user disable sunshine
# 启动 sunshine 服务
systemctl --user start sunshine
# 停止 sunshine 服务
systemctl --user stop sunshine

  登录进入 WebUI 管理界面后，可以在故障排除里查看启动日志，确定正常启动。这里贴上本机对于 sunshine 的简单配置(AI注解)：

cat .config/sunshine/sunshine.conf

## #################
## 网络绑定设置
## #################

## 局域网
## 服务监听地址：0.0.0.0 表示监听本机所有网络接口（有线、无线等）
## 允许局域网内任何设备通过本机IP地址连接
address = 0.0.0.0

## #################
## 音频设置
## #################

## 音频捕获源：指定系统音频输出作为串流音频源
## alsa_output.pci-0000_06_00.6.analog-stereo 是声卡设备标识
## .monitor 表示捕获的是该声卡的"监视器"源（即系统播放的声音，而非麦克风输入）
audio_sink = alsa_output.pci-0000_06_00.6.analog-stereo.monitor

## #################
## 视频捕获设置
## #################

## 获取屏幕
## 屏幕捕获方法：kms (Kernel Mode Setting)
## 使用Linux内核级直接渲染管理器进行底层屏幕捕获，不依赖X11或Wayland合成器
## 需要用户有video组权限，适用于无头或最小化环境
capture = kms

## 目标显示输出名称：指定要捕获的显示器DRM设备标识
## 这里的 '1' 是简写，通常对应 /sys/class/drm/card1-XXX 这样的设备
## 可通过 cat /sys/class/drm/card*/status 命令查看可用输出
output_name = 1

## 支持的分辨率列表：定义可提供给客户端的视频流分辨率
## 即使原生分辨率更高，也会缩放至此列表中的分辨率进行编码传输
## 格式为 [宽x高, 宽x高, ...]，当前只设置1080p一种选项
resolutions = [1920x1080]

## #################
## 编码器设置
## #################

## 视频编码器：nvenc (NVIDIA NVENC)
## 使用NVIDIA显卡的专用硬件编码器进行视频编码
## 优点：极高效率，极低延迟，几乎不占用CPU资源
## 要求：必须配备支持NVENC的NVIDIA显卡并安装官方驱动
encoder = nvenc

## #################
## 输入控制设置
## #################

## 控制器输入：disabled (完全禁用)
## 客户端无法使用手柄、游戏手柄等控制器设备反向控制主机
controller = disabled

## 键盘输入：disabled (完全禁用)
## 客户端无法使用键盘输入反向控制主机
keyboard = disabled

## 鼠标输入：disabled (完全禁用)
## 客户端无法使用鼠标输入反向控制主机
mouse = disabled

## #################
## 通用设置
## #################

## 本地化语言：zh (中文)
## 设置用户界面和日志显示的语言为中文
locale = zh

## 最低日志级别：2 (Warning/警告级别)
## 日志记录详细程度：0=Verbose/详细, 1=Debug/调试, 2=Info/信息, 3=Warning/警告, 4=Error/错误
## 设置为2表示只记录警告及以上级别的重要信息，减少日志输出量
min_log_level = 2

  当完全启动成功后，平板打开 moonlight 下拉刷新或者手动输入 IP 地址即可开启副屏之路。

四、问题

  看起来很简单，但是我还是折腾了好几个小时，周五从11点搞到1点半，受困于 capture 问题，周六晚上又折腾2个小时，1点半左右才弄好。弄完心情大爽！

Q1. wayland 启动 capture 异常：Unable to initialize capture method
A1. X11 之前启动时并没有这个问题，这里的问题是权限不足。给 Sunshine 添加权限 sudo setcap cap_sys_admin+p $(readlink -f $(which sunshine)) 重启 Sunshine 即可。

Q2. Unable to initialize capture method Platform failed to initialize
A2. 添加权限后发现没有找到任何可用的显示输出(此时幸好我显卡诱骗器带EDID)，如果没有 EDID 的话，可以看下其他博客。强制KMS捕获 capture = kms

# 1. 查看显示器识别
xrandr --listmonitors
Monitors: 2
 0: +*eDP-2 2560/381x1440/214+0+0  eDP-2
 1: +HDMI-A-1 2400/344x1350/195+2560+0  HDMI-A-1

# 2. 确认输出口名称
ls /sys/class/drm/

# 3. 检查 EDID 是否有数据
sudo hexdump -C /sys/class/drm/card1-HDMI-A-1/edid

Q3. Error: Failed to create client: Daemon not running(Sunshine 的后台守护进程没起来)
A3. systemctl enable avahi-daemon

Q4. 防火墙问题
A4. 首先确定是否在同一网段，然后确保开启主机的防火墙，下面脚本梭哈。(还有可能是路由器开启AP隔离了哟)

# 正在使用的防火墙应用
# 检查 firewalld 是否运行
systemctl is-active firewalld
# 检查 UFW (Uncomplicated Firewall) 是否运行
systemctl is-active ufw
# 检查 iptables 服务（较少见，但可以检查）
systemctl is-active iptables
# 检查 nftables 服务（更新的工具，在某些系统上取代iptables）
systemctl is-active nftables

# 本机使用的是 firewalld 其他自行搜索使用
sudo firewall-cmd --permanent --add-port=47984-48010/tcp
sudo firewall-cmd --permanent --add-port=47984-48010/udp
sudo firewall-cmd --permanent --add-port=5353/udp
sudo firewall-cmd --reload

五、结语

  目前体验 moonlight 客户端使用 1080P-60HZ/120HZ 没什么问题，不过当开启2K以上高码率时，发现存在画面撕裂的现象，使用 1080P-60HZ 延迟超低，仅2-3ms。(完了，生活作息乱了😭)
