AOSP 源码环境

AOSP(Android Open Source Project)是适用于不同规格设备的操作系统。点击跳转官方文档

一、目录结构

AOSP 是一个巨大的工程，下面是目录结构说明：

aosp (Android Open Source Project)
  |--- .repo            // 由 `repo` 工具管理的版本库元数据目录。包含一个清单文件 (.xml) 来定义如何组合数百个 Git 仓库以构建一个完整的 Android 系统。
  |--- abi              // 应用二进制接口（ABI）检查相关工具和配置，用于确保二进制兼容性。
  |--- art              // Android Runtime (ART) 源码。负责将 DEX 字节码编译成本地机器码并执行，是现代 Android 应用运行的核心虚拟机。
  |--- bionic           // Android 的 C 运行时库（libc）、动态链接器（linker）等。提供了符合 POSIX 标准的系统调用、线程、数学函数等实现，专为移动设备优化。
  |--- bootable         // 启动和更新相关的代码，包括 recovery（恢复模式）和 bootloader（引导加载程序）参考实现。
  |--- build            // AOSP 的构建系统核心，包括 Soong (替代旧的 Makefile) 和 Blueprint 文件。定义了如何编译整个 Android 系统。
  |--- cts              // Compatibility Test Suite (兼容性测试套件)。用于确保设备实现符合 Android 兼容性定义文档 (CDD) 的要求。
  |--- developers       // 为开发者提供的一些工具和示例代码。
  |--- development      // 高级开发工具和应用程序，例如 IDE 插件、系统调试工具等。
  |--- device           // 特定于硬件设备（如 Google Pixel, Nexus）的配置、内核和硬件抽象层 (HAL) 代码。通常由 OEM 厂商添加。
  |--- external         // 大量使用的第三方开源项目（库和工具）的源码，例如 OpenSSL, SQLite, FFmpeg, Clang/LLVM 等。
  |--- frameworks       // Android 框架层的核心代码，分为 Java 和 Native 两部分。
         |--- base      // Java 层的核心框架代码。Android 系统服务（如 ActivityManagerService, WindowManagerService, PackageManagerService）、API 和 UI 组件都在这里。
                |--- libs
                       |--- hwui          // 硬件加速的 2D 渲染引擎，负责 View 的绘制和动画的渲染（RenderThread 在此运行）。
         |--- native    // Native (C/C++) 层的框架代码，提供底层服务和支持。
                |--- services
                       |--- inputflinger    // 系统服务，处理来自全局的输入事件（触摸、按键）并将其分发给合适的窗口。
                       |--- surfaceflinger  // 系统合成服务，接收所有窗口的缓冲区，将它们合成并最终发送给显示设备。
  |--- hardware         // 硬件抽象层 (HAL) 接口定义和部分实现。提供了标准接口让 Android 系统与设备特定的硬件驱动进行通信。
  |--- kernel           // 预编译的 Linux 内核二进制文件。内核源码本身托管在 kernel.org 或其他硬件相关的仓库中（如 kernel/msm）。
  |--- libcore          // Apache Harmony 项目的衍生，提供了 Java 核心类库（如 java.*, javax.*）的实现，包括 Harmony 和 OpenJDK 的代码。
  |--- libnativehelper  // 提供 JNI（Java Native Interface）开发的辅助函数，简化了在 Native 代码中调用 Java 方法和转换数据类型的过程。
  |--- out              // 编译输出目录。包含所有的中间对象文件、最终的系统镜像（system.img, boot.img, recovery.img 等）。
  |--- packages         // 系统核心应用和关键服务模块，例如 Launcher（桌面）、SystemUI（状态栏）、Settings（设置）、Calendar（日历）、Camera（相机）等。
  |--- pdk              // Platform Development Kit (平台开发工具包)，用于帮助硬件制造商将新系统集成到设备上。
  |--- prebuilts        // 预编译好的各种工具链和二进制文件，如 GCC, Clang, NDK, SDK 等，以确保构建环境的一致性。
  |--- sdk              // Software Development Kit (软件开发工具包) 相关工具和模拟器皮肤等。
  |--- system           // Android 操作系统的核心部分。
         |--- core      // 系统核心工具和守护进程，对 Android 至关重要。
                |--- init   // `init` 进程的源码。这是内核启动后的第一个用户空间进程，负责解析 init.rc 脚本、启动系统守护进程和服务。
         |--- extras    // 一些额外的、可选的 Android 工具和库，如 `sqlite3` 命令行工具。
  |--- test             // 大型测试框架和测试用例，例如 VTS (Vendor Test Suite)。
  |--- tools            // 各种开发和支持工具。例如 `aidegen` (为 AOSP 模块生成 IDE 配置)、`emulator` 相关工具等。
  |--- vendor           // 厂商专有软件和硬件驱动目录。包含设备制造商（如 Google, Qualcomm）提供的闭源或特定硬件相关的库、固件和 HAL 实现。

二、下载

  按照官方文档操作，受到网络影响会失败，这里推荐拉取中科大的镜像，进行同步更新。操作文档

1、安装 repo

yay -S repo

2、基础包

  接着从中科大提供的打包好的 AOSP 镜像下载地址并解压、然后修改配置并同步。

tar xf aosp-latest.tar
cd aosp
git -C ./.repo/manifests.git/ remote set-url origin https://mirrors.ustc.edu.cn/aosp/platform/manifest.git

3、选择分支

  通过文档查看BuildID、分支标识、支持设备的列表。个人比较激进，使用最新的分支 android-latest-release

cd aosp
repo init -b android-latest-release
repo sync

  同步需要大量的时间，建议保持网络畅通。

4、下载驱动二进制文件(可选)

  一部分硬件驱动由硬件厂商提供，Google并没有这些驱动的源码。暂时没有折腾，只想玩一玩编译模拟模拟器镜像，看看代码。
  通过上一步查询到的 BuildID ，在此页面查找二进制文件并集成到源码即可。

二、编译

  • 初始化编译环境
source ./build/envsetup.sh
  • 选择编译产品(最新版本已经去除 lunch 选择项，更多参考文档)
# 使用 Cuttlefish emulator
lunch aosp_cf_x86_64_only_phone-aosp_current-eng 

# 使用 Android Emulator
lunch sdk_phone64_x86_64-ap3a-eng
  • 编译
m

三、源码查看

这里推荐使用 ASFP(android-studio-for-platform), 安装 yay -S android-studio-for-platform。调整 JVM 内存，可以参考IDEA开发配置

yay -S android-studio-for-platform

打开 ASFP 新建项目 -> 指定 aosp 路径 -> 完成等待缓存加载即可。文档

后续可以通过配置 asfp-config.json 添加模块进行同步，比如 frameworks/base,packages/apps/Settings
